# Colors

Quill's palette is grounded in aged paper and sepia ink — warm, editorial, and
intentional. The accent (**moss** by default, switchable per surface via
`data-accent`) carries interactive emphasis: links, focus rings, eyebrows.
Every value on this page renders live from `src/tokens/quill.tokens.mjs`
(Dawn cuts shown — each token re-cuts per theme).

## Papers

Background surfaces. Use in order from lightest to deepest for visual hierarchy.

<Swatch token="--paper" hex={c.paper.base.light} usage="page background, default surface" />
<Swatch token="--paper-warm" hex={c.paper.warm.light} usage="cards, popovers, sidebars" />
<Swatch token="--paper-deep" hex={c.paper.deep.light} usage="pressed states, secondary surfaces, icon containers" />

## Inks

Text and iconography. The three-step ink scale mirrors the hierarchy of type weight.

<Swatch token="--ink" hex={c.ink.base.light} usage="primary text, headings, icons" />
<Swatch token="--ink-soft" hex={c.ink.soft.light} usage="body text, secondary labels" />
<Swatch token="--ink-muted" hex={c.ink.muted.light} usage="placeholder, captions, disabled text" />

## Pigments

Used sparingly — one pigment per surface, never competing. Each `-deep` cut
carries hover/pressed and text duty (AA on paper); gold alone needs a darker
`-text` cut for AA text on light grounds.

<Swatch token="--terracotta" hex={c.pigment.terracotta.base.light} usage="danger, destructive; accent option" />
<Swatch token="--terracotta-deep" hex={c.pigment.terracotta.deep.light} usage="terracotta hover / pressed / text duty" />
<Swatch token="--moss" hex={c.pigment.moss.base.light} usage="success; the default accent" />
<Swatch token="--moss-deep" hex={c.pigment.moss.deep.light} usage="moss hover / pressed / text duty (links, rings)" />
<Swatch token="--indigo" hex={c.pigment.indigo.base.light} usage="info; accent option" />
<Swatch token="--indigo-deep" hex={c.pigment.indigo.deep.light} usage="indigo hover / pressed / text duty" />

<Swatch token="--gold-text" hex={c.pigment.gold.text.light} usage="gold text duty — the deep cut misses 4.5:1 on paper" />

## Hairlines

Borders and dividers. Ink at low alpha for passive lines; interactive controls
get the solid `control` boundary, which clears WCAG 1.4.11 non-text 3:1.

<BorderSwatch token="--line-faint" value={c.line.faint.light} usage="subtle dividers, ghost borders" />
<BorderSwatch token="--line-soft" value={c.line.soft.light} usage="default border (--border)" />
<BorderSwatch token="--line" value={c.line.base.light} usage="stronger rules" />
<BorderSwatch token="--line-strong" value={c.line.strong.light} usage="emphasis borders" />
<BorderSwatch token="--line-control" value={c.line.control.light} usage="interactive control boundaries (--input): fields, switch tracks, checkboxes" />

## Data visualization

Chart-only cuts — the UI pigments read gray as data marks and fail
colorblind-safety checks, so charts get their own palette, validated per theme
(CVD ΔE ≥ 8 between neighbors, ≥ 3:1 on every ground). **Assign series colors
in this fixed order, never cycled or reordered** — a filter that drops a series
must not repaint the survivors.

<Swatch token="--chart-series-1" hex={c.chart.series[1].light} usage="series 1 — terracotta cut (--chart-1)" />
<Swatch token="--chart-series-2" hex={c.chart.series[2].light} usage="series 2 — indigo cut (--chart-2)" />
<Swatch token="--chart-series-3" hex={c.chart.series[3].light} usage="series 3 — gold cut (--chart-3)" />
<Swatch token="--chart-series-4" hex={c.chart.series[4].light} usage="series 4 — plum, a chart-only hue (--chart-4)" />
<Swatch token="--chart-series-5" hex={c.chart.series[5].light} usage="series 5 — moss cut (--chart-5)" />

Magnitude and polarity use ramps, not the series palette:

<Ramp token="--chart-seq-1 … 5" steps={c.chart.seq} note="sequential (magnitude): one hue — moss — low → high emphasis; heatmaps, choropleths" />
<Ramp token="--chart-div-1 … 5" steps={c.chart.div} note="diverging (polarity): terracotta ↔ indigo around a neutral midpoint — never red/green" />
